# Historia de Usuario – HU-05

## Gestión básica de reservas

**Como** usuario del sistema
**quiero** poder registrar una reserva de una cancha desde una interfaz web
**para** apartar un horario disponible y evitar conflictos con otras reservas.

---

# Descripción

El sistema permitirá a los usuarios registrar reservas de canchas mediante un formulario web.
La solicitud será enviada al microservicio de reservas, el cual validará la disponibilidad del horario y almacenará la información en la base de datos.

En caso de existir una reserva en el mismo horario, el sistema deberá retornar un error indicando que el horario ya se encuentra ocupado.

---

# Criterios de aceptación

### Creación de reserva

Dado que un usuario quiere reservar una cancha
Cuando envía la información de la reserva
Entonces el sistema debe guardar la reserva en la base de datos.

---

### Validación de disponibilidad

Dado que ya existe una reserva para una cancha en un horario específico
Cuando otro usuario intenta reservar la misma cancha en el mismo horario
Entonces el sistema debe mostrar un error indicando que el horario no está disponible.

---

### Visualización desde el frontend

Dado que el usuario utiliza el formulario de reservas
Cuando envía los datos
Entonces la interfaz debe mostrar:

* mensaje de éxito si la reserva fue creada
* mensaje de error si el horario ya está ocupado
